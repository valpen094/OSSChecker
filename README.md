# OSSChecker

OSSCheckerは、[OSV.dev](https://osv.dev) APIを使用して、様々なエコシステム（NuGet, npm, PyPIなど）のオープンソースパッケージの既知の脆弱性をチェックするコマンドラインツールです。

## 機能
- **マルチエコシステム対応**: NuGet, npm, PyPI, Maven, Go, crates.io, RubyGems, Packagist, Pub に対応。
- **インタラクティブ検索**: パッケージを検索し、候補から選択可能。
- **脆弱性レポート**: CVE、深刻度、影響を受けるバージョンを含む詳細なレポートを作成。

## レポート生成仕様

### 保存先
生成されたすべてのレポートは、アプリケーションのルートフォルダにある `Reports` ディレクトリに保存されます。

### 生成ロジック
チェック実行後、以下のルールに基づいてレポートファイル（Markdown形式）の生成を試みます：

1.  **パッケージが見つかった場合 (ヒット)**:
    -   パッケージ検索で有効な候補が見つかった場合（パッケージが存在する場合）、レポートは**生成されます**。
    -   **脆弱性あり**: 脆弱性が見つかった場合、その詳細がレポートに記載されます。
    -   **安全 (Safe)**: 脆弱性が0件の場合でも、"Status: SAFE" というメッセージと共にレポートが生成されます。

2.  **パッケージが見つからない場合 (ヒットなし)**:
    -   パッケージ検索で候補が見つからなかった場合（無効なパッケージ名など）、レポート生成は**スキップされます**。
    -   コンソールに "Package source not found (no search hits)" と表示されます。

## 使用方法

1.  アプリケーションを実行します:
    ```bash
    dotnet run
    ```
2.  操作を選択します（検索/チェック または エコシステムの切り替え）。
3.  パッケージ名を入力します。
4.  出力を確認し、詳細なMarkdownレポートについては `Reports` フォルダを確認してください。
